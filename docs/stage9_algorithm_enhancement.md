# 阶段 9：Terrain-aware 遮挡算法增强计划

## 目标

1. 修复现存遮挡序列问题：混合索引导致的 `1970-01-01` 占位时间、缺失功率/辐照度值、pvlib Sandia 逆变器参数不兼容。
2. 引入更精细的遮挡判定：根据坡度、行间几何与太阳向量计算遮挡阈值，减少误判。
3. 评估性能并为后续阶段（3D 可视化、数据接口优化）提供更可靠的遮挡基础数据。

## 操作步骤

1. **问题复现与数据分析**
   - 复查 `shading_1.json` 中的 `1970` 时间戳，定位遮挡序列构建逻辑（`TerrainBacktrackingSolver`）。
   - 统计各时段 `shading_multiplier` 与天气数据的对齐情况，确认功率/辐照度为何为 `null`。
   - 复现 Sandia 参数报错，确认 pvlib 版本支持的参数集合。

2. **算法设计**
   - 定义遮挡判定所需的几何量：行间距、坡度差、太阳高度/方位。
   - 选择射线判定方案（自实现或基于 `trimesh`/`pyembree`），并设计简化版（投影计算）作为回退。
   - 明确性能目标（例如：720 条记录、403 行内完成在 1 s 内）。

3. **实现与调试**
   - 重构 `TerrainBacktrackingSolver.compute_tracker_angles`，确保返回 `pd.Series` 时保留原索引、避免整数混入。
   - 在遮挡乘数输出中写入同步的功率/辐照度（可在调用 `calculate_pv_power` 前缓存原始数据）。
   - 调整逆变器模型调用：
     - 优先尝试 pvlib 支持的参数格式；
     - 或改用 `pvlib.inverter.pvwatts` 等稳健模型。

4. **性能评估**
   - 对比增强前后的耗时、内存、遮挡乘数差异。
   - 记录潜在并行化或缓存（例如邻接矩阵重用）改进点。

5. **验证与文档**
   - 撰写 `docs/stage9_notes.md`，记录问题、解决方案与测试命令。
   - 更新单元/集成测试，覆盖遮挡索引、乘数范围、逆变器输出。
   - 在执行计划中标记阶段完成情况并总结成果。
