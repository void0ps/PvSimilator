## 后端启动

### 步骤 1：进入后端目录
```bash
cd backend
```

### 步骤 2：创建虚拟环境（如果尚未创建）
```bash
# Windows
python -m venv venv

# Linux/Mac
python3 -m venv venv
```

### 步骤 3：激活虚拟环境
```bash
# Windows PowerShell
.\venv\Scripts\Activate.ps1

# Windows CMD
.\venv\Scripts\activate.bat

# Linux/Mac
source venv/bin/activate
```

### 步骤 4：安装依赖
```bash
pip install -r requirements.txt
```

**注意**：如果遇到某些包安装失败，可以尝试简化版依赖：
```bash
pip install -r requirements_simple.txt
```

### 步骤 5：启动后端服务
```bash
python main.py
```

或者使用 uvicorn 直接启动：
```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 步骤 6：验证后端
打开浏览器访问：
- API 文档: http://localhost:8000/docs
- 健康检查: http://localhost:8000/health

**预期输出**：
```
INFO:     Started server process [xxxxx]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

## 前端启动

### 步骤 1：进入前端目录
```bash
cd frontend/analysis
```

### 步骤 2：安装依赖
```bash
npm install
```

或使用 pnpm（更快）：
```bash
pnpm install
```

**注意**：如果遇到依赖冲突，尝试：
```bash
npm install --legacy-peer-deps
```

### 步骤 3：启动开发服务器
```bash
npm run dev
```

### 步骤 4：验证前端
打开浏览器访问：
- 前端应用: http://localhost:3000

**预期输出**：
```
  VITE v4.4.5  ready in 1234 ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
  ➜  press h to show help
```

## 常见问题

### ❌ 问题 1：后端启动失败 - 端口被占用
**错误信息**：
```
ERROR: [Errno 10048] error while attempting to bind on address ('0.0.0.0', 8000)
```

**解决方案**：
```bash
# 方法1：修改端口
uvicorn main:app --reload --port 8001

# 方法2：查找并终止占用进程（Windows）
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# 方法2：查找并终止占用进程（Linux/Mac）
lsof -ti:8000 | xargs kill -9
```

---

### ❌ 问题 2：前端启动失败 - 依赖冲突
**错误信息**：
```
npm ERR! peer dependency conflict
```

**解决方案**：
```bash
# 清理缓存重装
rm -rf node_modules package-lock.json
npm install --legacy-peer-deps

# 或使用 yarn
yarn install
```

---

### ❌ 问题 3：后端无法连接数据库
**错误信息**：
```
sqlalchemy.exc.OperationalError: unable to open database file
```

**解决方案**：
确保 `backend/pv_simulator.db` 存在，或让程序自动创建：
```bash
cd backend
python -c "from app.core.database import Base, engine; Base.metadata.create_all(bind=engine)"
```

---

### ❌ 问题 4：后端缺少 pandas/numpy 等包
**错误信息**：
```
ModuleNotFoundError: No module named 'pandas'
```

**解决方案**：
```bash
# 确认虚拟环境已激活（命令行前应有 (venv) 标识）
# 重新安装依赖
pip install pandas numpy pvlib scipy openpyxl
```

---

### ❌ 问题 5：前端访问后端 API 失败（CORS 错误）
**错误信息**：
```
Access to XMLHttpRequest blocked by CORS policy
```

**解决方案**：
检查 `backend/main.py` 的 CORS 配置，确保包含前端地址：
```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "http://127.0.0.1:3000"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

---

### ❌ 问题 6：找不到地形数据文件
**错误信息**：
```
FileNotFoundError: 未找到地形数据文件: 带坡度地形数据.xlsx
```

**解决方案**：
确保项目根目录存在 `带坡度地形数据.xlsx` 文件，或在 API 调用时上传新的数据文件。

## 生产环境部署

### 后端生产部署
```bash
cd backend

# 使用 gunicorn（推荐）
pip install gunicorn
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000

# 或使用 uvicorn
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

### 前端生产构建
```bash
cd frontend/analysis

# 构建生产版本
npm run build

# 预览构建结果
npm run preview

# 生产文件位于 dist/ 目录
# 可使用 nginx 或其他 web 服务器部署
```
