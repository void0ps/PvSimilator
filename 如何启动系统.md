# 🚀 如何启动 PV Simulator 系统

## 快速启动（推荐）

### 方法1：使用启动脚本

#### Windows PowerShell（推荐）

**终端1 - 启动后端：**
```powershell
.\启动后端.ps1
```

**终端2 - 启动前端（新开一个终端）：**
```powershell
.\启动前端.ps1
```

#### Windows CMD

**终端1 - 启动后端：**
```cmd
启动后端.bat
```

**终端2 - 启动前端：**
```cmd
cd frontend\analysis
npm run dev
```

---

## 手动启动（详细步骤）

### 步骤1：启动后端

#### 打开第一个终端（PowerShell）

```powershell
# 1. 进入项目根目录
cd D:\code\PvSimilator

# 2. 进入后端目录
cd backend

# 3. 激活虚拟环境
.\venv\Scripts\Activate.ps1

# 4. 启动服务
python main.py
```

**预期输出：**
```
INFO:     Started server process [xxxxx]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
```

✅ 后端启动成功！

---

### 步骤2：启动前端

#### 打开第二个终端（PowerShell）

```powershell
# 1. 进入项目根目录
cd D:\code\PvSimilator

# 2. 进入前端目录
cd frontend\analysis

# 3. 启动开发服务器
npm run dev
```

**预期输出：**
```
  VITE v4.4.5  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

✅ 前端启动成功！

## 常见问题

### ❌ 问题1：后端启动失败

**错误：`ModuleNotFoundError: No module named 'fastapi'`**

**解决：**
```powershell
cd backend
pip install -r requirements.txt
```

---

### ❌ 问题2：端口被占用

**错误：`Address already in use`**

**查找并终止占用进程：**
```powershell
# 查找占用8000端口的进程
netstat -ano | findstr :8000

# 终止进程（替换<PID>为实际进程ID）
taskkill /PID <PID> /F
```

---

### ❌ 问题3：前端依赖缺失

**错误：`Cannot find module 'xxx'`**

**解决：**
```powershell
cd frontend\analysis
npm install
```

---

### ❌ 问题4：虚拟环境无法激活

**错误：`无法加载文件...因为在此系统上禁止运行脚本`**

**解决：**
```powershell
# 以管理员身份运行PowerShell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

## 生产环境部署

### 后端（使用Gunicorn）
```powershell
cd backend
pip install gunicorn
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
```

### 前端（构建静态文件）
```powershell
cd frontend\analysis
npm run build
# 静态文件生成在 dist/ 目录
```
